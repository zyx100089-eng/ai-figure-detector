# Real or Rendered? Detecting AI-Generated Scientific Figures

Can we tell the difference between real scientific figures from published papers and figures generated by AI? This project investigates detection methods across two categories of AI-generated scientific figures:

- **Pixel-level fakes**: Figures generated directly by image models (Stable Diffusion)
- **Code-level fakes**: Figures produced by executing LLM-generated plotting code (GPT-4o)

We compare three detection approaches and evaluate which generation methods are hardest to detect.

## Key Results

| Detector | Accuracy | F1 Score | AUC-ROC |
|---|---|---|---|
| CNN (ResNet-18) | 98.5% | 0.988 | 1.000 |
| Statistical + RF | 97.3% | 0.972 | 0.996 |
| Frequency + SVM | 90.8% | 0.930 | 0.978 |

**Key finding**: Pixel-level fakes (Stable Diffusion) are harder to detect than code-level fakes (GPT-4o matplotlib), reducing F1 by up to 3% across detectors.

## Setup

```bash
git clone https://github.com/zyx100089-eng/ai-figure-detector.git
cd ai-figure-detector
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Reproducing Results

### 1. Collect real figures from arXiv
```bash
PYTHONPATH=. python src/data_collection/arxiv_scraper.py
PYTHONPATH=. python src/data_collection/figure_extractor.py
PYTHONPATH=. python src/data_collection/figure_filter.py
```

### 2. Generate fake figures
```bash
# Code-level fakes (requires OpenAI API key)
export OPENAI_API_KEY=your-key
PYTHONPATH=. python src/generation/generate_llm_code.py -n 200

# Pixel-level fakes (runs locally, no API key needed)
PYTHONPATH=. python src/generation/generate_sd.py -n 50
```

### 3. Build dataset and train detectors
```bash
PYTHONPATH=. python src/data_collection/build_dataset.py
PYTHONPATH=. python src/detection/cnn_detector.py
PYTHONPATH=. python src/detection/frequency_detector.py
PYTHONPATH=. python src/detection/stat_detector.py
```

### 4. Generate paper figures
```bash
PYTHONPATH=. python src/experiments/generate_plots.py
```
